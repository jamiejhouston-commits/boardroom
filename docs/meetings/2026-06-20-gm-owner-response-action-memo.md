## Parking app — revised GM direction

Core job: help a driver avoid parking uncertainty and help an operator monetize/organize parking spaces with trustworthy status, timing, and payment records.

### Feature option 1: Parking Pin + Meter Timer With Proof Trail

- User problem: drivers forget where they parked, lose track of meter time, or cannot prove when/where they parked.
- Proposed behavior: one-tap parking pin, countdown timer, extension reminders, photo/context note, and a local proof trail containing timestamp, approximate location, and optional receipt/photo.
- Why it is not just Apple Maps: the value is not only remembering location; it is the combined parking session record, timer, receipt/photo context, and reminder workflow.
- User value: 5/5
- Marketing value: 4/5
- Revenue potential: 3/5
- Engineering effort: Small/Medium
- QA risk: Medium because location/timer accuracy must be tested.
- MVP version: pin + timer + notification + photo note + session history.
- Success signal: users create repeat parking sessions and return to view history/reminders.

### Feature option 2: Shared Safe Parking Notes

- User problem: people do not know if a parking spot is safe, legal, tow-risky, or locally understood.
- Proposed behavior: users can privately save or publicly contribute simple notes: safe after hours, paid zone, tow risk, entrance hint, lighting concern, accessibility note.
- Why it is not just a map: this is local parking intelligence, not navigation.
- User value: 4/5
- Marketing value: 4/5
- Revenue potential: 4/5 if bundled into premium/local-city packs or operator listings.
- Engineering effort: Medium
- QA risk: Medium/High due to moderation and incorrect claims.
- MVP version: private notes first; public/shared notes later after moderation policy.
- Success signal: users save notes and reopen them at future parking sessions.

### Feature option 3: Operator Micro-Lot Mode

- User problem: small property owners/churches/gyms/events may have temporary parking capacity but no lightweight tool to manage it.
- Proposed behavior: owner creates a micro-lot, sets capacity, opening hours, pricing/free status, and shares a simple link/QR code.
- Why it is not covered by Apple: Apple helps drivers navigate; this helps small operators publish/manage availability.
- User value: 4/5
- Marketing value: 5/5
- Revenue potential: 5/5
- Engineering effort: Large if payments are included; Medium without payments.
- QA risk: High if real-time availability or payments are included.
- MVP version: no payments; availability listing + QR/share + manual capacity updates.
- Success signal: one real operator can use it for an event without staff training.

## Recommended sequence

1. Quick win MVP: Parking Pin + Meter Timer With Proof Trail.
2. Differentiator: Private Safe Parking Notes.
3. Commercial Phase 2: Operator Micro-Lot Mode after MVP validation.
